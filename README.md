The personal website of **Djalma Jr.** — about, portfolio, blog and CV. It is a
sibling to my engineering-focused site, [djalmajr.dev](https://djalmajr.dev).

Built with **Astro 6** and **Tailwind CSS 4**, fully internationalized (English,
Portuguese and Spanish) and deployed to **Cloudflare Pages**.

## Features

- **Internationalization** with URL-prefixed locales: English at `/` (default),
  Portuguese at `/pt`, Spanish at `/es`. UI chrome is translated; content is
  shared. The sitemap includes `hreflang` alternates.
- **Content-driven**: bio, CV, projects and posts all live in Markdown under
  `src/content/`. Bio and CV have one Markdown file per locale (`en`, `pt`, `es`).
- **Light / dark themes**, an RSS feed (also localized) and a tag-based content
  index.
- **Fast and minimal**: static output, near-zero client-side JavaScript.

## Tech stack

- [Astro](https://astro.build/) 6 (static output, `astro:i18n`)
- [Tailwind CSS](https://tailwindcss.com/) 4 (via `@tailwindcss/vite`)
- TypeScript
- [Bun](https://bun.sh/) as the package manager and task runner

## Getting started

This project uses **Bun** (Node.js >= 22.12.0).

```bash
bun install
bun run dev      # start the dev server at http://localhost:4321
```

## Scripts

| Command                | Action                                            |
| :--------------------- | :------------------------------------------------ |
| `bun run dev`          | Start the local dev server                        |
| `bun run build`        | Type-check (`astro check`) and build to `./dist/` |
| `bun run preview`      | Preview the production build locally              |
| `bun run lint`         | Run ESLint                                        |
| `bun run format`       | Format all files with Prettier                    |
| `bun run format:check` | Check formatting without writing                  |

## Project structure

```text
src/
├── config/        # Site, navigation, social and page configuration
├── content/       # Markdown content
│   ├── bio/       # en.md / pt.md / es.md
│   ├── cv/        # en.md / pt.md / es.md
│   ├── posts/
│   └── projects/
├── i18n/          # Locale definitions, UI strings and helpers
│   ├── locales.ts # Locales, DEFAULT_LOCALE, NonDefaultLocales
│   ├── types.ts   # UIStrings interface
│   └── lang/      # en.ts / pt.ts / es.ts
├── layouts/       # Page layout wrappers
├── pages/         # Routes; `[locale]/` holds thin localized wrappers
└── styles/        # Global CSS
```

## Internationalization

- Default locale (`en`) is served at the root with no prefix.
- Other locales (`pt`, `es`) are served under their prefix and generated by the
  thin wrappers in `src/pages/[locale]/`, which reuse the root page components.
- UI strings live in `src/i18n/lang/{en,pt,es}.ts` and are read via
  `useTranslations(Astro.currentLocale)`. Internal links use
  `getRelativeLocaleUrl`.

## Deployment

The site is deployed to **Cloudflare Pages**. `bun run build` produces static
output in `./dist/`, which Cloudflare serves directly (see `wrangler.toml`).
Deployment runs from GitHub Actions on pushes to `main` (`.github/workflows/`).
